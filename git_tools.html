Before you dive in, ensure Git and GitHub CLI are installed and configured.

üîπ 2. Starting a Repository

Learn how to initialize a new Git repository locally or directly on GitHub.

Create a repo locally:

git init               # Initializes a new Git repository in the current directory
git add .              # Stages all changes in the current directory
git commit -m "Initial commit" # Commits the staged changes with a message

Create repo on GitHub directly from CLI:

gh repo create my-repo-name --public --source=. --remote=origin --push
# This command does three things:
# 1. Creates a new public repository on GitHub named 'my-repo-name'.
# 2. Links your local repository to this new GitHub repo (sets 'origin' remote).
# 3. Pushes your current local branch to the 'main' branch on GitHub.

üëâ Tip: This is a powerful command that streamlines the process of getting your local project onto GitHub!

üîπ 3. Daily Git Workflow

The core commands you'll use every day for managing your code.

Stage ‚Üí Commit ‚Üí Push:

git status              # Shows modified, staged, and untracked files
git add file.py         # Stages specific file(s) for the next commit
git add .               # Stages all changes in the current directory (new, modified, deleted)
git commit -m "Message" # Records staged changes to the repository with a descriptive message
git push origin main    # Uploads local commits to the remote repository's 'main' branch

Pull changes from GitHub:

git pull origin main    # Fetches changes from the 'main' branch of the 'origin' remote and merges them into your current branch

üîπ 4. Branching (Very Important)

Branches allow you to work on new features or bug fixes in isolation.

git branch              # Lists all local branches, highlighting the current one
git checkout -b feature-login   # Creates a new branch named 'feature-login' and switches to it
git checkout main                # Switches to the 'main' branch
git merge feature-login          # Integrates changes from 'feature-login' into the current branch (e.g., 'main')
git branch -d feature-login      # Deletes the local branch 'feature-login' (only after it's merged)

With GitHub CLI for Pull Requests:

gh pr create   # Interactively creates a new pull request on GitHub
gh pr status   # Shows the status of your current pull requests
gh pr merge    # Merges a pull request directly from the CLI

üîπ 5. Seeing History

Explore the commit history of your repository.

git log                  # Displays a detailed log of all commits
git log --oneline        # Shows a concise, single-line summary of each commit
git log --graph --oneline --all   # Visualizes the commit history as a graph, including all branches

üîπ 6. Switching to a Previous Version

How to navigate through your project's history safely.

Checkout a commit temporarily:

git checkout <commit-id> # Switches to a specific commit. This puts you in a "detached HEAD" state.

Reset branch to an old commit (‚ö†Ô∏è Destructive):

git reset --hard <commit-id> # WARNING: This rewrites history and discards all changes after the specified commit. Use with extreme caution!

Create a new branch from old commit (Safe way!):

git checkout -b old-version <commit-id> # Creates a new branch named 'old-version' starting from the specified commit-id.
# This allows you to explore previous states without affecting your main branches.

üëâ Tip: Always prefer creating a new branch when you want to look at or work from an old commit, as `git reset --hard` is destructive.

üîπ 7. Undoing Changes

Commands to revert or modify unwanted changes.

Undo changes in a file (before staging):

git checkout -- file.py # Discards unstaged changes in 'file.py', reverting it to the last committed state.

Unstage a file:

git reset file.py       # Removes 'file.py' from the staging area, keeping its changes in your working directory.

Amend last commit (fix typo, etc.):

git commit --amend -m "New message" # Combines staged changes with the previous commit and allows you to change its message.
# Useful for fixing small issues without creating a new commit.

üîπ 8. GitHub CLI Superpowers

Leverage the GitHub CLI for common repository interactions.

  • Clone repo: 
    gh repo clone owner/repo # Clones a repository from GitHub to your local machine.
  • View repo in browser: 
    gh repo view --web # Opens the current repository's page in your default web browser.
  • List issues: 
    gh issue list # Lists open issues in the current repository.
  • Create issue: 
    gh issue create # Interactively creates a new issue.
  • List pull requests: 
    gh pr list # Lists open pull requests in the current repository.

üîπ 9. Tags (for Versions / Releases)

Mark specific points in your history, usually for releases.

git tag v1.0             # Creates a lightweight tag named 'v1.0' at the current commit.
git push origin v1.0     # Pushes the tag 'v1.0' to the 'origin' remote.

With GitHub CLI (Release Management):

gh release create v1.0 --notes "First stable release" # Creates a new GitHub release associated with tag 'v1.0'.
gh release list        # Lists releases in the current repository.
